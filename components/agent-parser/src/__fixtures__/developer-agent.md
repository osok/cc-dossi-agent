## Behaviour

The Developer implements code based on design documents. It follows established patterns from existing components. When invoked by Task Manager, it receives a task description and creates or modifies source files.

## Inputs

- Design document
- Task description
- Convention files

## Outputs

- Source code files
- Updated configuration
- Git commits

## Success Criteria

- [ ] Code compiles without errors
- [ ] All requirements addressed
- [ ] Follows existing patterns
